# PRODUCTION serving deps for the grading ORACLE worker (oracle_worker.py).
#
# The exact tiers (stud 7th-street, post-last-draw draw) are PURE-PYTHON stdlib —
# they need NOTHING here (that is why prod historically ran ORACLE_PYTHON=python3
# torch-free). The ONLY runtime dep is numpy, required solely by the CERTIFIED-NET
# pre-last-draw badugi path (net_forward_numpy.NumpyValueNet — a torch-free numpy
# forward over nets/badugi_draw1.npz). Torch is NOT a serving dep: the net ships as
# a plain .npz and is served on numpy alone (Option A). Keep this minimal — one
# wheel — so the Render build stays fast and small.
#
# Deploy: point the oracle worker's python (ORACLE_PYTHON) at an interpreter with
# this installed, e.g.  pip install -r solver/neural/requirements-serving.txt
# If numpy is absent the net path fails safe (blueprint fallback); the exact tiers
# and stud are unaffected.
#
# numpy 2.x COMPATIBILITY: the net forward (net_forward_numpy.NumpyValueNet) is
# now numpy-1.x-AND-2.x clean (verified: f64 forward bit-identical across numpy
# 1.26 and 2.x to ~1e-14; no removed 2.x aliases; no ndim>0->scalar conversion,
# which numpy 2.4 made a hard TypeError). The old `<2` cap was the reason the net
# path silently fell back to the blueprint on Render's Python-3.14 venv, where
# ONLY numpy 2.x wheels exist (1.x cannot be installed) — so `<2` was
# unsatisfiable there. Allow 2.x so the pin matches the interpreter prod builds.
